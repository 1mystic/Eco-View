
# RUN 

Step 1 — Run the Kaggle Notebook (≈ 1 hour, free T4 GPU)
Go to kaggle.com/code → New Notebook → File → Import Notebook → upload notebooks/EcoView_KD_Notebook.ipynb
In the right panel → Data → Add Data → search trashnet → add TrashNet by garythung (2,527 images, 6 classes)
Settings → Accelerator: GPU T4 × 2
Run All — takes ~45–60 min. Saves 7 artifacts to /kaggle/working/
Key artifacts you need at the end:


student_model_quantized.onnx   ← the model (~6 MB)
class_mapping.json             ← {0: "cardboard", 1: "glass", ...}
Download both from the Kaggle output panel.

Step 2 — Update the Label Map
Open class_mapping.json. It will look like:


{"0": "cardboard", "1": "glass", "2": "metal", "3": "paper", "4": "plastic", "5": "trash"}
Edit ecoview-ml-backend/core/inference.py — update LABELS and SEVERITY_MAP to match exactly:


LABELS = ["cardboard", "glass", "metal", "paper", "plastic", "trash"]

SEVERITY_MAP = {
    "cardboard": "low",
    "glass":     "medium",
    "metal":     "medium",
    "paper":     "low",
    "plastic":   "high",
    "trash":     "high",
}
Step 3 — Add numpy + onnxruntime Back to Requirements
Edit ecoview-ml-backend/requirements.txt:


fastapi>=0.115
uvicorn[standard]>=0.30
httpx>=0.27.0
pillow>=10.3.0
python-dotenv>=1.0.0
numpy>=1.26.0
onnxruntime>=1.18.0
Step 4 — Push Model + Code to HF Space

# Copy updated files to HF Space repo
Copy-Item "ecoview-ml-backend\core\inference.py" "ecoview-ml\core\inference.py" -Force
Copy-Item "ecoview-ml-backend\requirements.txt"  "ecoview-ml\requirements.txt"  -Force

# Copy the model file (downloaded from Kaggle)
Copy-Item "path\to\student_model_quantized.onnx" "ecoview-ml\student_model_quantized.onnx"

cd ecoview-ml
git add core/inference.py requirements.txt student_model_quantized.onnx
git commit -m "feat: add ONNX student model + updated labels for TrashNet classes"
git push
HF Space rebuilds automatically (~2–3 min with numpy+onnxruntime install).

Step 5 — Verify ONNX Mode Is Live
Hit the health endpoint:


https://mozoj4-ecoview-ml.hf.space/health
You should see "mode": "onnx" instead of "mode": "simulator":


{
  "components": {
    "student_classifier": {
      "mode": "onnx",
      "model_path": "/app/student_model_quantized.onnx"
    }
  }
}
What Changes After This
Before (now)	After Step 5
Every report → Gemini API call	Reports ≥85% confident → ONNX only (~50ms)
1,500 Gemini calls/day limit	~85% of calls handled locally, Gemini only for edge cases
Labels: garbage_dump, plastic_waste…	Labels: cardboard, glass, plastic, trash…
Cold start risk if Gemini rate-limited	Resilient — ONNX runs with no API key
The only catch: TrashNet classes (cardboard, glass, plastic…) don't align with EcoView's display labels. Once you have real data, re-run the notebook with your own labelled pollution photos (from the flywheel at data/flywheel/) to train a domain-specific model with the original EcoView categories. That's v2 → v2.1.
